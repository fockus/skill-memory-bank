# Plan: feature — session-memory & MB hygiene gaps (A1–A6)

**Baseline commit:** 13684a831386fd36fa4b9f8df2475e89d2efa4ce

## Context

**Problem.** A live audit of a real consumer project (FaberlicApp) surfaced six gaps in the
Memory-Bank session-memory + tooling layer:
- **G6 / A1** — `scripts/mb-note.sh` writes notes with **no YAML frontmatter** → `index.json`
  gets empty `tags` / null `importance`; notes are not tag-filterable.
- **G5 / A2** — there is **no way to rebuild `_recent.md`** from existing session files; it is
  only ever updated incrementally on SessionEnd. After a cleanup it must be regenerated by hand.
- **G4 / A3** — `scripts/mb-compact.sh` compacts `plans/done` + `notes/` but **never touches
  `session/`**, which grows unbounded (53 stale files removed by hand during the audit).
- **G3 / A4** — recall ranking (`hooks/lib/semantic_store.py::search`) is pure cosine with **no
  source-type weighting**, so raw `transcript/*.jsonl` chunks out-rank curated `notes/`.
- **G2 / A5** — `mb-drift.sh` has no **plan-status vocabulary** validation; non-canonical
  `status:` values and roadmap↔plan status mismatch go undetected.
- **A6 (new)** — `mb-drift.sh::check_terminology` uses BSD-grep `\b` which **mis-handles Cyrillic
  word boundaries**, false-matching declined common nouns ("этапы", "фазы") and incidental prose,
  not just decomposition markers → noisy `terminology=warn`.

**Expected result.** Each gap closed behind tests, default behaviour preserved where it could
surprise existing users, every new behaviour given an off-switch. All changes in the canonical
skill repo (survive reinstall); then propagated to live copies.

**Related files:** `scripts/mb-note.sh`, `scripts/mb-compact.sh`, `scripts/mb-drift.sh`,
`hooks/mb-session-end.sh`, `hooks/lib/semantic_store.py`, `hooks/lib/searcher.py`,
new `scripts/mb-session-recent-rebuild.sh`, `tests/bats/*`, `hooks/tests/*`.

**Discipline.** TDD per stage (RED→GREEN), full `bats tests/bats/` + `pytest hooks/tests/` green
before each commit, propagate hooks to `~/.claude/skills/memory-bank/` + project `.memory-bank/bin/`,
one commit per stage, push on explicit go.

## Stages

<!-- mb-stage:1 -->
### Stage 1 (A1): mb-note.sh writes YAML frontmatter

**What to do:** make `scripts/mb-note.sh` emit a frontmatter block (`type: note`, `tags: []`,
`importance: medium`, `source: manual`) above the `# <title>` heading.

**DoD (SMART):**
- A note created by `mb-note.sh <topic>` starts with `---` and contains `type:`, `tags:`,
  `importance:`, `source:` keys.
- `mb-index-json.py` parses the new note with non-null `importance`.
- Existing behaviour (filename collision `_2`/`_3`, topic slug) unchanged.

**TDD:** new `tests/bats/test_note_frontmatter.bats` — RED first (asserts `---` first line), then
implement. **Verify:** `bats tests/bats/test_note_frontmatter.bats` green; full bats green.

<!-- mb-stage:2 -->
### Stage 2 (A2): rebuild `_recent.md` command

**What to do:** add `scripts/mb-session-recent-rebuild.sh <mb_path>` — scan `session/*.md`
(skip `_recent.md`), keep the newest `MB_RECENT_KEEP` (default 5) that have a `## Summary`,
emit the `## <date> <time> (<branch>) — <sid8>` + summary blocks newest-first. Wire a mention
into `commands/mb.md` (e.g. `/mb recall --rebuild` or a documented script call).

**DoD (SMART):**
- Given N session files (≥1 empty, ≥1 without `## Summary`), the script writes `_recent.md`
  with exactly the newest ≤KEEP substantive summaries, newest-first.
- Empty / summary-less sessions are excluded.
- Idempotent: running twice yields byte-identical output.

**TDD:** new `tests/bats/test_recent_rebuild.bats` (fixtures: empty, no-summary, 2 good) — RED,
then implement. **Verify:** bats green.

<!-- mb-stage:3 -->
### Stage 3 (A3): mb-compact.sh archives stale session files

**What to do:** extend `scripts/mb-compact.sh` to list `session/*.md` older than the decay
threshold (reuse existing age logic) as compaction candidates and, on `--apply`, move them to
`session/archive/` (never `_recent.md`, never sessions newer than threshold). Mirror the notes
archival pattern.

**DoD (SMART):**
- `--dry-run` reports stale session candidates with a count; `--apply` moves them to
  `session/archive/`, leaves `_recent.md` + fresh sessions in place; idempotent re-run = no-op.
- Off unless a file is genuinely older than the threshold.

**TDD:** extend `tests/bats/` (new `test_compact_sessions.bats`) — RED, then implement.
**Verify:** bats green.

<!-- mb-stage:4 -->
### Stage 4 (A4): recall source-type weighting

**What to do:** in `hooks/lib/semantic_store.py::search` (and/or `searcher.py`) apply a per-`kind`
score multiplier so curated `note`/`session` chunks out-rank raw `transcript` at comparable
cosine. Weights from `MB_RECALL_SOURCE_WEIGHTS` (default e.g. `note=1.0,session=0.95,transcript=0.85`);
unset/`off` ⇒ legacy pure-cosine order (back-compat).

**DoD (SMART):**
- With two chunks of equal cosine, a `note` ranks above a `transcript`.
- `MB_RECALL_SOURCE_WEIGHTS=off` reproduces the pre-change ordering exactly.
- No crash when `kind` is missing (treat as weight 1.0).

**TDD:** extend `hooks/tests/semantic_searcher_test.py` (or new) — RED, then implement.
**Verify:** `pytest hooks/tests/` green.

<!-- mb-stage:5 -->
### Stage 5 (A5): plan-status vocabulary drift check

**What to do:** define the canonical plan-status vocabulary (doc in `references/`), add a
`mb-drift.sh` checker that flags `plans/*.md` whose frontmatter `status:` is outside the
canonical set, and (best-effort) roadmap↔plan status mismatch. Deterministic only — prose-lie
detection stays out of scope (note it for `/mb verify`/`mb-doctor`).

**DoD (SMART):**
- A plan with `status: active` (non-canonical) → `drift_check_plan_status=warn`; canonical
  (`queued`/`in_progress`/`done`/`blocked`) → `ok`.
- Checker is additive; existing checkers unaffected; `drift_warnings` count reflects it.

**TDD:** extend `tests/bats/` (drift) — RED, then implement. **Verify:** bats green.

<!-- mb-stage:6 -->
### Stage 6 (A6): terminology checker — Unicode-correct boundaries

**What to do:** fix `mb-drift.sh::check_terminology` so it does not false-match Cyrillic declined
forms / incidental prose. Restrict matching to decomposition markers (heading lines `^#{1,6}\s*(Этап|…)`
and table/`mb-stage` contexts) and/or use a boundary that works for Cyrillic on BSD grep
(e.g. surrounding non-letter assertions / `grep -P` when available, with a portable fallback).

**DoD (SMART):**
- A plan containing only Russian prose nouns ("каждый этап", "эта фаза") → `terminology=ok`.
- A plan with a real Cyrillic decomposition header (`## Этап 1`) → `warn`.
- Existing English-marker plans stay `ok`.

**TDD:** extend `tests/bats/` (terminology) with prose-vs-marker fixtures — RED, then implement.
**Verify:** bats green; re-run drift on FaberlicApp drops `terminology` to ok.
